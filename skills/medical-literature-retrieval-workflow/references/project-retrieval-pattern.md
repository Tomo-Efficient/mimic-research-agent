# 项目工作流模式：EDA 到文献检索与选题

## 核心拆分

完整流程不是单纯“搜文献”，而是把用户数据和外部证据连接起来：

1. EDA 解析
   - 读取 EDA 报告。
   - 提取人群、疾病、暴露变量、结局变量、分组变量、检测方法。
   - 提取显著相关变量、样本量、事件数和主要限制。

2. PICO/PECO 构建
   - 用 EDA 中可验证的变量构建 PICO/PECO。
   - 暴露变量来自数据，不凭空发明。
   - 结局变量必须有清晰事件定义或连续指标定义。

3. PubMed 检索策略
   - 根据 PICO/PECO 生成 MeSH + 自由词查询。
   - 分三类检索：近 3 年原始研究、近 5 年综述、不限时间精确组合。
   - 新变量、新检测方法和缩写优先加入 `[Title/Abstract]`。

4. 文献证据结构化
   - 只保留真实可验证文献。
   - 每条文献记录 PMID、DOI、链接和证据类型。
   - 摘要研究对象、样本量、暴露、结局、方法、主要发现、局限性。

5. Evidence gap matrix
   - 比较 EDA 发现与已有文献证据。
   - 判断 gap 是人群、暴露、结局、方法、组合、验证还是机制 gap。
   - 不把“本轮未检出”写成“确定空白”。

6. Candidate idea scoring
   - 从 gap matrix 生成 5-10 个候选 idea。
   - 按创新性、数据可验证性、临床意义、统计可行性、发表潜力和风险评分。
   - 输出最推荐的 10 个 idea；不足 10 个时输出全部并说明限制。

## 检索策略原则

- 数据先行：idea 必须能被用户数据验证。
- 文献校准：idea 必须有足够背景文献，但直接组合证据不能过密。
- 先召回，后收窄：不要一开始过度过滤。
- 同一概念内用 `OR` 覆盖同义词、缩写、拼写变体、检测方法和旧名。
- 不同概念之间用 `AND` 形成精确组合。
- 时间窗动态计算：近 3 年和近 5 年以当前日期为准。

## PubMed 三类查询

| 查询 | 目的 | 时间窗 | 过滤 |
|---|---|---|---|
| 近 3 年原始研究 | 找近期直接证据和研究设计 | 当前日期回溯 3 年 | 排除 review，保留临床/观察/实验原始研究 |
| 近 5 年综述 | 判断主题成熟度和综述密度 | 当前日期回溯 5 年 | review/systematic review/meta-analysis |
| 不限时间精确组合 | 判断该变量组合是否已有直接历史证据 | 不限 | 精确 P + E/I + O 组合 |

## 数据库选择

| 数据库 | 主要用途 | 可能盲点 |
|---|---|---|
| PubMed | 医学核心检索、MeSH、PMID 核验 | 新兴预印本和部分会议摘要 |
| Embase | 药物、药理、不良反应和欧洲期刊 | 访问受限 |
| Web of Science | 引文网络和跨学科覆盖 | 无 MeSH |
| Google Scholar | 引用追踪和灰区文献 | 元数据噪声大 |
| Cochrane Library | 系统综述和临床试验证据 | 机制和早期转化研究弱 |
| ClinicalTrials.gov | 试验注册和进行中研究 | 不是已发表论文 |

## 候选文献记录规则

最低字段：
- Title
- Authors
- Year
- Journal/Venue
- Source Database
- Direct Link
- DOI status
- PMID status
- Evidence Status

DOI 或 PMID 不存在时，写 `not available`；没有核实时写 `not verified`。

## 常见错误

- 跳过 EDA 变量定义，直接凭主题生成 idea。
- 只看 p 值，不看事件数、效应量、缺失值和混杂因素。
- 把相关性写成因果。
- 把 PubMed 检索不到写成确定空白。
- 用模型记忆补论文信息。
- 把综述结论当成用户数据可验证假设。
